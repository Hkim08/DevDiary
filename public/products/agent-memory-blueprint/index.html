The Problem

Context collapse is the silent killer of production AI agents.

Every LLM processes a fixed context window on each API call. Nothing persists automatically between calls. By call #30, if you haven't actively managed what enters the window, your agent is operating completely blind — generating responses based on assumptions instead of facts, repeating actions it already took, ignoring the constraints you defined at the start.

This is not a Claude problem. Not a GPT-4o problem. Not a Gemini problem. It is a structural property of how every large language model works, and it will catch every builder who hasn't built the architecture to manage it.

It presents in four recognizable patterns:

Signature A
The Amnesiac
Agent asks the user to repeat information already provided mid-task. Prior turns dropped from the window with no summarization.
→ "Could you remind me what your goal was?"
Signature B
The Confabulator
Agent references decisions, names, or facts that were never provided. Context gaps filled with confident hallucination.
→ References "the approach we agreed on" (you never agreed)
Signature C
The Loop
Same tool call repeated 3+ times with no state change. No working memory tracking what has already been attempted.
→ Calls search_docs() identically for the 4th time
Signature D
The Identity Drift
Named-persona agent stops following its role constraints after 20+ turns. System prompt evicted from the window.
→ "Rex the analyst" becomes generic helpful assistant

500 daily sessions at a 15% collapse rate is 75 failures a day. At even a 10% refund conversion, that's 7–8 refunds before you account for churn. Fix the architecture once. It saves continuously.

The Solution

Five strategies. One framework.
Zero ambiguity.

The Blueprint maps every context management approach — from a 10-line sliding window to a full episodic memory system — to a decision framework that tells you exactly which one fits your situation.

01
Sliding Window
Keep the N most recent turns. Simplest to implement. Right for tasks under 10 turns where early context is non-critical.
⚠ Breaks when: session exceeds 20 turns or early instructions are critical
02
Progressive Summarization
Compress older turns into a dense narrative on a trigger cadence. Preserves semantic content at a fraction of the token cost.
⚠ Breaks when: 5+ compression cycles degrade precision below usable threshold
03
Structured State
Maintain a typed JSON state object updated after every turn. Model reads state, acts, updates state. Explicit and auditable.
⚠ Breaks when: state schema grows unbounded and model starts skipping fields
04
External Retrieval (RAG)
Store everything externally. Retrieve only semantically relevant chunks per call. Required for knowledge-heavy or multi-session agents.
⚠ Breaks when: retrieval quality degrades without a reranking step
05
Episodic Memory
Persist structured episode records between sessions. Load the 3 most relevant at session start. Enables genuine cross-session coherence.
⚠ Breaks when: no pruning strategy — episode volume and retrieval latency grow unbounded

The Blueprint's decision framework answers four questions about your agent and maps the answers directly to a recommended strategy — no guesswork.

Anti-Patterns

The 10 things builders do
that guarantee collapse.

Every one of these is a real pattern observed in production. The Blueprint maps each one to an observable signal — what you see when it's happening — and a specific fix from the implementation templates.

AP-01
Dumping the Full History
Passing every turn verbatim until the window overflows, then truncating from the front. The model loses its initial instructions first — exactly the worst thing to lose.
Fix → Progressive Summarization (PT-02) + keep last N turns verbatim
AP-02
The Lazy System Prompt
"You are a helpful assistant." Works for a demo. Collapses by turn 15 as the model drifts toward its default behavior with nothing to anchor it.
Fix → PT-01 Session Init with role, constraints, and task framing
AP-03
Trusting Model Memory
Assuming the model remembers something because you said it earlier. It does — until the window fills. There is no memory without architecture.
Fix → Tier 2 state object tracks everything critical; never trust implicit recall past turn 10
AP-04
State in Prose
Tracking state as narrative paragraphs: "The user decided in turn 3 and changed their mind in turn 7..." Prose state is expensive and the model misreads it constantly.
Fix → State belongs in typed JSON. Prose is for context narrative. Never mix them.
AP-05
Giant State Objects
State JSON that grows to 4KB+ with every field ever touched. The model starts skipping fields. You pay tokens for data the model ignores.
Fix → State holds only CURRENT data. Archive completed steps. Hard cap at ~900 tokens.
AP-06
Compression Without a Budget
Compressing context with no word count target. The model writes 600 words. You save 50 tokens. You lose the structural integrity of the summary.
Fix → PT-02 specifies 150–200 words maximum. Enforce it in the prompt.
AP-07
Cold Handoffs
Passing a task from Agent A to Agent B with only the original task description. Agent B has no knowledge of what A tried, learned, or decided. It starts from scratch.
Fix → PT-05 Handoff Brief + create_handoff_payload() in context_manager.py
AP-08
Persona Without Anchoring
Defining a named agent persona once and never re-injecting it. For sessions over 20 turns, persona constraints decay. The specialist becomes a generic assistant.
Fix → PT-06 anchors at turns 15, 30, 45. Automate with should_inject_anchor().
AP-09
No Failure Detection
No mechanism to detect looping, confabulating, or drifting. Session runs to completion. Output is garbage. You find out when the user complains.
Fix → Monitor for: identical tool calls in succession, turn count > 2× expected
AP-10
RAG Without Reranking
Semantic similarity ≠ task relevance. Top-5 retrieval gives the model a related-but-wrong chunk. It acts on it confidently.
Fix → Retrieve top-10, ask the model to select the 3 most relevant before using them.
